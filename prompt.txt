SECTION 3 — TOOLS TO IMPLEMENT (tools.py + schemas.py)
═══════════════════════════════════════════════════════════

Implement ALL 5 tools below as real working Python functions.
Each tool must have a proper JSON schema in schemas.py.

TOOL 1: get_weather(city: str, unit: str = "celsius")
  - Call the free Open-Meteo API (no key required):
    https://geocoding-api.open-meteo.com/v1/search?name={city}
    then https://api.open-meteo.com/v1/forecast with lat/lon
  - Return: city, temperature, unit, condition, humidity, wind_speed
  - Schema: city (string, required), unit (enum: celsius/fahrenheit)

TOOL 2: calculate(expression: str, precision: int = 2)
  - Safely evaluate math expressions using Python's ast module
    (NEVER use eval() directly — parse the AST and only allow
     Num, BinOp, UnaryOp, and the operators +,-,*,/,**,%)
  - Return: expression, result (rounded to precision), steps (list 
    of intermediate operations shown as strings)
  - Schema: expression (string, required), precision (int, default 2)

TOOL 3: search_dictionary(word: str, language: str = "en")
  - Call the free DictionaryAPI: 
    https://api.dictionaryapi.dev/api/v2/entries/{language}/{word}
  - Return: word, phonetic, part_of_speech, definitions (top 3),
    example_sentences (top 2), synonyms (top 5)
  - Schema: word (string, required), language (string, default "en")

TOOL 4: get_current_time(timezone: str = "UTC")
  - Use Python's datetime + zoneinfo (stdlib, no external dep)
  - Return: timezone, current_time (ISO 8601), unix_timestamp,
    day_of_week, is_weekend, utc_offset
  - Handle invalid timezone with a clear error message
  - Schema: timezone (string, default "UTC")

TOOL 5: unit_converter(value: float, from_unit: str, to_unit: str)
  - Support these category groups:
      LENGTH: mm, cm, m, km, inch, foot, yard, mile
      WEIGHT: mg, g, kg, tonne, ounce, pound
      TEMPERATURE: celsius, fahrenheit, kelvin
      SPEED: mps, kph, mph, knot
  - Return: original_value, original_unit, converted_value,
    converted_unit, category, formula_used
  - Raise ToolError if units are from different categories
  - Schema: value (number), from_unit (string), to_unit (string)

SECTION 7 — AGENT CLASS (agent.py)
═══════════════════════════════════════════════════════════

class ToolCallingAgent:

  __init__(self, model="claude-opus-4-5", max_tokens=4096, 
           max_tool_rounds=10):
    - Load ANTHROPIC_API_KEY from .env
    - Initialize anthropic.Anthropic() client
    - Store model, max_tokens, max_tool_rounds
    - Initialize conversation_history: list[dict]

  SYSTEM PROMPT (set as a constant in agent.py):
    You are a helpful, precise AI assistant with access to tools 
    for weather, math, dictionary lookups, time, and unit conversion.
    Always use tools when the user's request requires real data.
    When you call a tool and receive results, synthesize the data 
    into a natural, informative response. If a tool returns an error,
    explain the issue clearly and suggest alternatives. Never fabricate 
    data that a tool should provide.

  run(self, user_message: str) -> AgentResponse:
    Implement the FULL agentic tool-calling loop:

    1. Append user_message to conversation_history
    2. Send to Claude with tools=ANTHROPIC_TOOLS
    3. Check stop_reason:
       - "end_turn" → extract text, return AgentResponse
       - "tool_use" → enter tool execution loop:
           a. Extract all tool_use blocks from response.content
           b. For EACH tool_use block:
              - Parse tool name + input
              - Validate input matches schema (raise SchemaValidationError if not)
              - Call the real Python function with a timer
              - Catch all exceptions → handle_tool_error()
              - Build tool_result content block
           c. Append assistant message (with tool_use blocks) to history
           d. Append user message with tool_result blocks to history
           e. Call Claude again → loop back to step 3
       - Increment round counter; if > max_tool_rounds raise AgentError
    4. Track all ToolCall instances throughout the loop
    5. Return fully populated AgentResponse

  reset_conversation(self):
    - Clear conversation_history

  get_conversation_summary(self) -> dict:
    - Return stats: total turns, total tool calls, tools used
